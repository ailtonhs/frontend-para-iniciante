Padrões e Melhores Práticas

Um elemento HTML pode ser alvo de estilização de diferentes maneiras. Por exemplo, um elemento pode ter tanto uma classe quanto um ID.

        
            <p id="text1" class="info"> Texto </p>

Aqui estão 3 formas de mirar o mesmo elemento HTML:

  • Seletor de elemento: p
  • Seletor de ID: #text
  • Seletor de classe: .info

Cada tipo de seletor CSS tem seu lugar nesta lista de prioridades. Quando várias regras CSS visam o mesmo elemento HTML, o navegador usa esta várias reras CSS visam o mesmo elemento HTML, o navegador usa esta ordem para decidir qual regra aplicar.

  1. Estilo inline
  2. IDs
  3. Classes
  4. Elementos

Esta ordem de prioridade é conhecida como especificidade. A especificidade é o algoritmo usado pelos navegadores para determinar a regra a ser aplicada a um elemento.

Você pode adicionar notas ou esplicações ao seu cógigo CSS. Comentários ajudam outras pessoas (e você no futuro!) a ler seu código.

Comentários em CSS começam com /* e termina com */.

        
            <style>

                /*Regra para o id box1*/

                #box1 {
                background-color: green;
                }
            </style>

Os Comentários CSS precisa estar contidos em tags <style> quando se usa CSS interno.

CSS externo

Uma alternativa ao CSS inline e interno é o CSS externo. O código CSS externo é escrito fora do documento HTML, em um arquivo separado.

Uma das vantagens do CSS extrno é que o mesmo arquivo de estilo CSS pode ser usado por vários documentos HTML(ou páginas da web).

Principais Conclusões da Lição

  • Especificidade indica ao navegador qual regra aplicar
  • Comentários em CSS começam com /* e termina com */
  • CSS externo significa um arquivo .css separado para as regras de estilo
